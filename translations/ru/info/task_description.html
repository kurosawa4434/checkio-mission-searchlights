Сколько освещённых вершин правильных многоугольников ты видишь?

В качестве входных параметров даются два списка (в первом - многоугольники, во втором - окружности освещённых зон)

Каждый многоугольник задаётся кортежем, содержащим 4 целых числа:

  • координату верхней вершины по оси X;
  • координату верхней вершины по оси Y;
  • длину ребра многоугольника;
  • количество вершин многоугольника.

Каждая окружность зоны освещённости задаётся кортежем, содержащим 3 целых числа:

  • координату центра окружности по оси X;
  • координату центра окружности по оси Y;
  • радиус окружности.

В качестве ответа необходимо вернуть общее количество освещённых вершин всех данных многоугольников.

Некоторые уточнения:

  • у всех заданных многоугольников их оси симметрии являются вертикальными линиями;
  • вершины, хотя бы одна координата которых является отрицательным числом, не учитывать при подсчёте (например (-1, 2), (2, -3), (-3, -4));
  • в тестах не будет вершин, лежащих на окружности или находящихся в непосредственной её близости

Примеры: 

searchlights([(2, 3, 2, 3)], [(1, 2, 1)]) == 1 # regular triangle
searchlights([(4, 5, 2, 4)], [(4, 4, 3)]) == 4 # square

Входные данные: два аргумента:

  • список, содержащий кортежи с 4-мя целочисленными параметрами многоугольников;
  • список, содержащий кортежи с 3-мя целочисленными параметрами окружностей.

Выходные данные: Количество освещённых вершин (не считая тех, хотя бы одна координата которых является отрицательной)

Предварительные условия:

  • длины ребёр многоугольников ≥ 1;
  • радиус окружностей зон освещения ≥ 1;
  • не будет случаев, когда вершины 2-х или более многоугольников совпадают (повторяются)
